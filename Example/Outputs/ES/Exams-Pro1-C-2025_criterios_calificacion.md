## Rúbrica de Calificación - Ejercicio 1: Seguimiento Semanal de Estudio

Esta rúbrica detalla los criterios de evaluación para el ejercicio de seguimiento semanal de estudio. La puntuación total es de 10 puntos.

**1. Corrección Funcional (4 puntos)**

*   **4 puntos:** El programa funciona correctamente y cumple con todos los requisitos especificados. La matriz se carga correctamente, se imprime correctamente y la función `sumRow` calcula el total de horas de estudio por asignatura de forma precisa. No hay errores de ejecución ni resultados incorrectos.
*   **3 puntos:** El programa funciona en su mayoría, pero presenta errores menores. Por ejemplo, la matriz se carga e imprime correctamente, pero la función `sumRow` tiene un error de cálculo en algunos casos.
*   **2 puntos:** El programa tiene errores significativos que impiden su correcto funcionamiento. Por ejemplo, la matriz no se carga correctamente o la función `sumRow` no devuelve resultados precisos.
*   **1 punto:** El programa no se ejecuta o no cumple con los requisitos mínimos.
*   **0 puntos:** No se presenta el programa o el programa no tiene ninguna funcionalidad.

**2. Calidad y Estructura del Código (3 puntos)**

*   **3 puntos:** El código está bien estructurado, es fácil de leer y entender. Se utilizan nombres de variables descriptivos y las funciones están bien definidas y cumplen con su propósito. Se evita la redundancia de código.
*   **2 puntos:** El código es funcional, pero la estructura podría mejorarse. Por ejemplo, los nombres de las variables no son descriptivos o las funciones son demasiado largas y complejas.
*   **1 punto:** El código es difícil de leer y entender. La estructura es deficiente y hay mucha redundancia.
*   **0 puntos:** El código es ilegible o no tiene estructura.

**3. Eficiencia y Optimización (2 puntos)**

*   **2 puntos:** El código es eficiente y utiliza los recursos de manera óptima. No hay operaciones innecesarias ni algoritmos ineficientes.
*   **1 punto:** El código funciona, pero podría ser más eficiente. Por ejemplo, se utilizan bucles innecesarios o se realizan cálculos redundantes.
*   **0 puntos:** El código es ineficiente y utiliza los recursos de manera deficiente.

**4. Estilo y Documentación (1 punto)**

*   **1 punto:** El código está bien documentado con comentarios claros y concisos que explican el propósito de cada función y sección del código. El código sigue un estilo de codificación consistente y legible.
*   **0 puntos:** El código no está documentado o la documentación es insuficiente. El estilo de codificación es inconsistente o ilegible.
